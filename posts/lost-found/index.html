mysql空磁盘挂卷问题

忽略lost+found文件

May 11, 2022
阅读时长: 1 分钟

今天在tke上部署mysql pod，以为很简单。结果发现pod一直crash，查看日志发现是挂卷的问题： [ERROR] --initialize specified but the data directory has files in it. Aborting.

解决办法很简单，给mysql加上启动参数--ignore-db-dir=lost+found即可。

mysql 5.7

k8s

name: mysql-master
image: mysql:5.7
args:
  - "--ignore-db-dir=lost+found"

docker compose

version: '3'
services:
  mysql-master:
    image: mysql:5.7
    command: [--ignore-db-dir=lost+found]
    environment:
      - MYSQL_ROOT_PASSWORD=root
kubernetes volume mysql 5.7
